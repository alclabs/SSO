Single Sign On Sample

The SSO Add-On demonstrates the use of server operators and privileges to control access to your add-on. Additionally it demonstrates linking back and forth between the server and your add-on while only presenting log in credentials a single time.

Try it out

Deploy the SSO sample add-on by executing the 'deploy' task and starting (if necessary) the server.

Browse to http://yourserver/sso (open on localhost). This should present a login page. Log in with any valid operator and password.

You can follow the links on the bottom of the page to link into the server without re-entering your password. The link to the protected page will only work if you logged in as an operator that has the "Access User Category 1" privilege.

Important Lessons

This web app add-on is a good example of basic security configuraiton in a WEB-INF/web.xml file. It contains two different security-constraints: one requires a valid operator for access to any page (by requiring the login role), the other requires a specific role (view_user_category_1) in order to access the /second.jsp page.

This add-on also provides good sample login form pages in the /loginout directory. Feel free to copy the /loginout directory and the WEB-INF/web.xml file as a starting point for your own Add-On.
